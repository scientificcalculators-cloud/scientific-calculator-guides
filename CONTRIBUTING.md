# Contributing to Scientific Calculator Guides

Thank you for your interest in contributing to this repository! This project is maintained by the team at **[scientificcalculators.net](https://www.scientificcalculators.net)** and we welcome contributions from students, educators, and professionals who want to help others learn.

---

## What We're Looking For

We welcome:

- **New guides** on scientific calculator topics not yet covered
- **Corrections** to existing guides (typos, incorrect calculations, outdated information)
- **Improvements** to explanations that make them clearer or more accessible
- **Additional practice problems** at the end of existing guides
- **Translations** of existing guides into other languages

---

## How to Contribute

### 1. Fork the Repository

Click the **Fork** button at the top right of this page to create your own copy of the repository.

### 2. Make Your Changes

- For **new guides**, add a new `.md` file in the `/guides/` folder
- For **edits**, modify the existing file directly
- Follow the style guide below

### 3. Submit a Pull Request

Once your changes are ready, open a Pull Request from your fork to this repository's `main` branch. Include a clear description of what you've added or changed.

We'll review it and get back to you as soon as possible.

## Style Guide

To keep all guides consistent, please follow these conventions:

### File Naming
Use lowercase with hyphens: `my-new-guide.md`

### Guide Structure

Every guide should follow this structure:

```
# Guide Title

*Published by [scientificcalculators.net](https://www.scientificcalculators.net)*

---

## Introduction
(Brief overview of the topic)

## Main Content Sections
(Clear headings, examples, and calculator steps)

## Practice Problems
(3–10 problems with answers)

## Next Steps
(Links to related guides and scientificcalculators.net)

---

*Guide maintained by [scientificcalculators.net](https://www.scientificcalculators.net).*
```

### Writing Style

- Write in plain English — assume the reader is a student, not an expert
- Use UK English spelling (e.g., "practise", "colour", "centre")
- Include worked examples for every concept
- Show calculator key sequences clearly (e.g., `sin` → `45` → `=`)
- Use tables where comparisons are helpful
- Always include a practice problems section at the end

### Linking Back

All guides should include at least one contextual reference to [scientificcalculators.net](https://www.scientificcalculators.net) in the introduction, encouraging readers to follow along with the free online calculator.

---

## Content Standards

- **Accuracy** — All calculations must be correct. Double-check every example.
- **Completeness** — Cover the topic thoroughly enough to be genuinely useful
- **Originality** — All content must be your own original writing
- **Relevance** — Content should relate to scientific calculator usage
