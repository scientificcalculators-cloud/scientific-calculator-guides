## Angle Mode: DEG vs RAD

**This is the most important setting to check before using trig functions.**

- **DEG** (Degrees) — use for everyday maths, GCSE, and most A-Level work
- **RAD** (Radians) — use for university-level maths, calculus, and physics

To switch: tap the **"Switch to Radians"** button below the keypad, or tap the DEG/RAD indicator at the top of the display.

> **Example of why this matters:**
> - `sin(30)` in DEG mode = **0.5** ✓
> - `sin(30)` in RAD mode = **−0.988** ✗ (wrong if you meant 30 degrees)

## Basic Arithmetic

Enter numbers and operations exactly as you'd write them:

- **Addition:** `5 + 3 =` → 8
- **Subtraction:** `10 − 4 =` → 6
- **Multiplication:** `6 × 7 =` → 42
- **Division:** `15 ÷ 4 =` → 3.75
- **Decimal:** use the `.` button — e.g. `3.14`

> **Note:** Dividing by zero will show an error — this is mathematically correct behaviour.

---

## Order of Operations

The calculator follows standard mathematical order (BODMAS):

1. Brackets
2. Powers and roots
3. Multiplication and division (left to right)
4. Addition and subtraction (left to right)

**Example:** `3 + 4 × 2 =` → **11** (not 14 — multiplication happens first)

To override order, use brackets: `(3 + 4) × 2 =` → **14**

## Powers and Roots

### Squaring
Press the number, then `x²`
Example: `9 x²` → 81

### Any Power
Press the base, then `x^y`, then the exponent, then `=`
Example: `2 x^y 8 =` → 256

### Square Root
Press `√` then the number
Example: `√ 144 =` → 12

### Cube Root
Press `∛` then the number
Example: `∛ 27 =` → 3

## Factorial (n!)

Calculates n × (n−1) × (n−2) × ... × 1. Works with non-negative integers only.

Example: `5 n! =` → 120 (5 × 4 × 3 × 2 × 1)

---

## Percentage (%)

Divides the number by 100.

Example: `75 % =` → 0.75

---

## Memory Functions — Step by Step

Memory is useful when you need to store an intermediate result and use it later.

**Example:** Calculate (12 × 5) + (8 × 3) using memory:

1. `12 × 5 =` → 60 → press `M+` (stores 60)
2. `8 × 3 =` → 24 → press `M+` (adds 24, memory now holds 84)
3. Press `MR` → display shows **84**

The memory value is always shown above the main display so you know what's stored.

## Tips for Best Results

- Always press **AC** to clear before starting a new calculation
- Always check **DEG/RAD** mode before using sin, cos, or tan
- Use **brackets** to control order of operations precisely
- Use **memory** (M+/MR) to store intermediate results in multi-step calculations
- If you get an unexpected answer, check your angle mode first — it's the most common cause of errors
